Présentation des épreuves E4 et E5

Epreuve E4: Analyse et Conception d'un Système d'Information

L'épreuve E4 du BTS SIO vise à évaluer votre capacité à analyser et à concevoir un système d'information (SI) répondant aux besoins d'une organisation.

Objectifs

  • Analyser les besoins et les contraintes d'une organisation en matière d'information
  • Définir les spécifications fonctionnelles et techniques du SI
  • Choisir les solutions techniques appropriées
  • Concevoir l'architecture du SI
  • Rédiger la documentation technique du SI

Déroulement de l'épreuve

L'épreuve E4 se déroule sur 8 heures et se compose de deux parties:

  • **Partie 1 (4 heures):** Étude de cas et réalisation d'une maquette du SI
  • **Partie 2 (4 heures):** Questionnement oral sur l'étude de cas et la maquette

Contenu de l'épreuve

L'épreuve E4 porte sur les domaines suivants:

  • Méthodes d'analyse et de conception de SI
  • Modèles de données
  • Langages de modélisation (UML)
  • Architectures réseaux
  • Solutions techniques de base de données
  • Solutions techniques de développement
  • Sécurité des SI

Conseils pour réussir

  • Maîtriser les méthodes d'analyse et de conception de SI
  • Savoir utiliser les langages de modélisation (UML)
  • Avoir des connaissances solides en architecture réseaux
  • Connaître les solutions techniques de base de données et de développement
  • S'intéresser à la sécurité des SI
  • S'entraîner à réaliser des études de cas et des maquettes de SI
  • Exerciter son oral en répondant à des questions sur des études de cas

Epreuve E5: Développement d'une Application Web ou Mobile

L'épreuve E5 du BTS SIO vise à évaluer votre capacité à développer une application web ou mobile répondant aux besoins d'une organisation.

Objectifs

  • Concevoir et réaliser l'interface utilisateur (IHM) de l'application
  • Développer la partie serveur de l'application
  • Mettre en œuvre la base de données de l'application
  • Déployer et tester l'application
  • Rédiger la documentation technique de l'application

Déroulement de l'épreuve

L'épreuve E5 se déroule sur 8 heures et se compose de deux parties:

  • **Partie 1 (6 heures):** Réalisation de l'application
  • **Partie 2 (2 heures):** Questionnement oral sur l'application et son code source

Contenu de l'épreuve

L'épreuve E5 porte sur les domaines suivants:

  • Langages de développement web (HTML, CSS, JavaScript)
  • Langages de développement côté serveur (PHP, Python, Java)
  • Bases de données relationnelles (SQL)
  • Interface de programmation d'applications (API)
  • Méthodologies de développement agile
  • Tests d'applications web et mobiles

Conseils pour réussir

  • Maîtriser les langages de développement web (HTML, CSS, JavaScript)
  • Avoir des connaissances en développement côté serveur (PHP, Python, Java)
  • Connaître les bases de données relationnelles (SQL)
  • Savoir utiliser les API
  • Comprendre les méthodologies de développement agile
  • Maîtriser les tests d'applications web et mobiles
  • S'entraîner à développer des applications web et mobiles
  • Exerciter son oral en répondant à des questions sur les applications web et mobiles

Mon projet E5

Projet Tardigrade: Réseau social pour la gestion des retards

Dans le cadre de l'épreuve E5, j'ai réalisé un projet Tardigrade, un réseau social en utilisant le framework Symfony, avec un accent particulier sur la création et la gestion de retards.

Fonctionnalités principales

  • Création de retards avec des détails (date, heure, raison)
  • Gestion des retards (modification, suppression)
  • Interface conviviale et intuitive
  • Mise en œuvre des fonctionnalités avancées de Symfony

Technologies utilisées

  • Framework Symfony
  • Langage de programmation PHP
  • Base de données MySQL
  • Langages web HTML, CSS et JavaScript

Compétences développées

  • Programmation orientée objet (POO)
  • Autonomie
  • Développement d'interfaces utilisateur
  • Travail en groupe

Résultats obtenus

J'ai réussi à développer un réseau social fonctionnel permettant la création, la gestion et la visualisation des retards. L'application utilise les fonctionnalités avancées de Symfony pour garantir la sécurité et les performances du système.
